Engineering Highlights

Key System Features & Innovation

Built with modern software engineering principles for reliability, speed, and security.

Interleaving Anti-Cheat Engine

Interleaves students department-wise (A, B, C, A, B...) in a continuous grid layout across rooms to prevent exam malpractice.

Pandas Batch Data Import

High-performance Excel (.xlsx) and CSV batch ingestion powered by `pandas`. Validates records, handles encodings, and reports line errors.

Strict DB Constraints

Enforces Django `UniqueConstraint` on database level. Prevents double-booked seats, duplicate student seats, or double-booked faculty.

Dynamic Hall Capacity Engine

Calculates hall capacities dynamically using `Rows × Columns` grid properties without redundant database fields.

Bulk Transaction Queries

Uses `transaction.atomic()` and `bulk_create()` to execute thousands of seat assignments in milliseconds with zero N+1 query bottlenecks.

Instant CSV Attendance Reports

Invigilators mark student presence in real time and generate structured downloadable CSV attendance reports for administrative record-keeping.

How The System Operates

1
Hall & Data Setup

2
Smart Allocation

The algorithm interleaves multi-dept students and assigns seats continuously across halls.

3
Invigilation Duty

Teachers view their hall duty, mark live student attendance, and export attendance sheets.

4
Student Self-Lookup

Students log in to check their hall location and seat code (e.g. S14) instantly.
